# 博客文章创作任务流程

本文档记录博客文章协同创作、自动创作与任务成文规则，适用于 `src/App.jsx`{应用状态中枢，负责文章创作页面切换、任务上下文和退出拦截}、`src/services/blogArticleAiStore.js`{文章创作任务存储服务，负责 localStorage 任务读写和状态更新}、`src/components/ui/Toast.jsx`{全局提示组件，负责顶部提示和文字操作}。

## 成文规则

- 创建任务只生成任务记录，不创建博客文章。
- 只有用户点击 `保存并编辑`{将任务产物写入文章列表并打开编辑器的操作} 后，任务才会成为文章。
- 是否已经成文以 `task.content.savedArticleId`{任务内容字段，保存已生成文章的 ID} 判断。
- 删除任务只删除任务记录，不删除已通过 `保存并编辑` 创建的文章。

## 创作模式

- `mode: 'collaborative'`{协同创作任务模式，用户按策划、大纲、正文阶段逐步确认} 使用协同创作多步骤页面。
- `mode: 'auto'`{自动创作任务模式，系统在自动创作过程页连续完成策划、大纲和正文} 使用自动创作查看过程页。
- `stage: 'auto-generating'`{自动创作运行状态，表示任务正在覆盖全部阶段生成} 代表自动任务正在生成。
- `stage: 'content-completed'`{内容完成状态，表示任务已生成可保存的文章终稿} 代表任务生成成功。

## 协同创作未保存退出

协同创作任务已创建但尚未保存成文章时，用户在策划页、大纲页或内容页点击退出，会被 `App.jsx` 拦截并显示确认弹窗。

固定提示文案：

```text
当前任务还未保存，是否退出创作流程？
```

固定操作：

- `退出`：返回博客文章页，保留任务记录，不创建文章。
- `自动完成`：调用 `convertCollaborativeTaskToAuto(projectId, taskId)`{任务转换工具，把同一条协同任务改为自动生成任务并保留已生成阶段数据}，把当前任务转换为自动创作任务。
- `继续任务`：关闭弹窗，停留在当前协同创作页面。

按钮从左到右为：白色 `自动完成`、红色 `退出`、蓝色 `继续任务`。

## 自动完成转换

`自动完成` 不新建任务，也不创建文章。转换后任务写回：

- `mode: 'auto'`
- `stage: 'auto-generating'`
- 清空 `errorMessage`{任务错误字段，用于标记失败原因}
- 保留已有 `planning`、`outline`、`content` 阶段数据

转换成功后返回博客文章页的任务列表，清空任务筛选条件，滚动并高亮当前任务，同时显示成功 Toast：

```text
已转为自动创作任务
```

## 提示与定位规则

- 单操作提示继续使用 `actionLabel/onAction`{Toast 兼容字段，用于既有单按钮提示}。
- 多操作提示使用 `actions`{Toast 多按钮数组，用于流程型提示的多个文字操作}。
- 未保存退出属于流程型决策，使用三操作确认弹窗，不使用 Toast。
- `自动完成` 成功后通过 `blogArticleNotice`{跨页面通知对象，携带提示文案、目标 tab 和任务 ID} 返回任务列表。
- 任务列表根据 `taskId` 清空筛选、滚动并高亮目标任务卡片，高亮约 2.6 秒后自动移除。
