# AGENTS.md

本文件是 AI Agent 在 Content Studio 仓库内工作的入口规则。目标是让人和 AI 都能快速理解项目边界，并且不破坏当前可运行状态。

## 先读顺序

1. `README.md`：确认项目定位、运行命令和目录职责。
2. 当前要修改目录下的 `README.md`：例如演示数据先读 `demo-data/rejin-cnc/README.md`。
3. `DESIGN.md`：涉及 UI、Figma、组件、图标、样式时必须阅读。
4. `docs/component-guidelines.md`：涉及共享组件和页面组件时必须阅读。

## 运行安全

- 不移动 `src/` 和 `demo-data/`，除非同步更新所有引用并通过 `pnpm build`。
- 不恢复用户已删除或已转移的文件，除非用户明确要求。
- 不把 `dist/` 当作源码修改。
- 不直接把页面组件里的本地持久化逻辑散落到组件内；新增持久化优先放到 `src/services/`。
- 修改 `src/`、运行时数据、构建配置、依赖、脚本或入口文件后，必须执行 `pnpm build`。
- 如果代码改动影响页面显示、交互、默认数据或生成内容，交付前还必须在浏览器验证最新内容可见。
- 如果浏览器仍显示旧内容，优先保证正常预览；可强制刷新、加时间戳访问，必要时使用 `pnpm dev:fresh` 或重启 preview。
- 纯文档、Figma 原型、设计规范说明、README、AGENTS 规则类改动，不强制执行 `pnpm build` 和浏览器验证；交付时说明这是文档-only 或原型-only 改动。

## 主动确认规则

开发过程中，如果原型图中的文案、按钮位置、按钮顺序、交互状态、数据展示或页面跳转逻辑存在模糊点，不要自行猜测实现；应先列出模糊点、给出推荐处理方式，并向用户确认。

如果用户的新开发任务与现有已开发内容、`DESIGN.md`、`docs/component-guidelines.md` 或当前目录边界存在冲突，不要直接覆盖或删除现有实现；应先说明冲突位置、影响范围和可选方案，并等待用户确认后再实施。

## 目录边界

- `src/`：应用源码。
- `demo-data/`：演示数据库，面向产品、内容和测试；业务内容保持英文。
- `docs/`：项目文档和规范索引。
- `scripts/`：支持脚本，运行前先读 `scripts/README.md`。

## UI And Figma

- 生成原型图或实现页面前，先按 `DESIGN.md` 使用 token。
- 组件优先复用 `src/components/ui/`。
- 图标统一使用 `lucide-react`。
- 不新增营销站式 hero、装饰性大渐变、无关插画或大面积单色装饰。
- Figma 标注必须使用可编辑文字，不要只用截图说明交互。
